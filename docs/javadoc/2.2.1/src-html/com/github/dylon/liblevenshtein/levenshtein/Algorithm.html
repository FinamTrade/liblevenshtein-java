003/**
004 * Variants of Levenshtein distance supported by this library.
005 * @author Dylon Edwards
006 * @since 2.1.0
007 */
008public enum Algorithm {
009
010  /**
011   * This is the standard, textbook version of Levenshtein distance. It includes
012   * support for the elementary operations of insertion, deletion and
013   * substitution.
014   */
015  STANDARD,
016
017  /**
018   * Sometimes known as Damerauâ€“Levenshtein distance, this includes the
019   * elementary operations described by the standard algorithm, as well as the
020   * elementary operation, transposition, which would incur a penalty of two
021   * units of error in the standard algorithm (whether it be an insertion and
022   * deletion, a deletion and insertion, or two substitutions). This algorithm
023   * is particularly useful in typed spelling-correction, where most spelling
024   * errors occur when a user transposes two characters.
025   */
026  TRANSPOSITION,
027
028  /**
029   * This algorithm includes the elementary operations described by the basic
030   * algorithm, as well as the elementary operations, merge and split.  A merge
031   * occurs when two characters are joined together, and a split occurs when a
032   * single character is exploded into two characters. This algorithm is
033   * particularly useful in OCR applications, where one may read a "cl" when one
034   * should have read a "d" (i.e. merge), or when one may have read a "d" when
035   * one should have read a "cl" (i.e. split).
036   */
037  MERGE_AND_SPLIT;
038}
